# codex-agents v0.1.1

## Summary

`v0.1.1` is a compatibility-focused follow-up release that migrates the bundled agent set from Markdown baselines to Codex-native `.toml` custom agents while keeping the CLI surface and managed entrypoint behavior stable.

## Highlights

- replaces bundled `agents/*.md` files with Codex-native `agents/*.toml`
- rewrites agent instructions into a shorter, execution-focused style aligned with Codex custom-agent conventions
- updates the source-managed install, status, doctor, list, and uninstall flows to operate on `.toml` agents
- adds a repository test that validates bundled agents are valid TOML with required Codex fields
- keeps the managed `AGENTS.md` orchestration entrypoint and `orchestrator-routing` skill behavior intact

## User Impact

- source installs from the repository now place `.toml` agents into `~/.codex/agents/`
- the global command surface remains the same:
  - `install`
  - `update`
  - `status`
  - `doctor`
  - `uninstall`
  - `list`
  - `version`
  - `link`
  - `unlink`
- users should still open a fresh Codex conversation after install or update so the latest agent and skill inventory is visible
