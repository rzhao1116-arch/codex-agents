# codex-agents v0.1.0

## Summary

`v0.1.0` is the first versioned public release of `codex-agents`: a Codex-oriented role and routing bundle that installs:

- a Markdown-based agent role set
- the `orchestrator-routing` skill
- a managed `AGENTS.md` entrypoint block
- a lifecycle CLI for install, update, status, doctor, uninstall, linking, and version reporting

## Highlights

- installs a curated set of specialist agents plus `orchestrator`
- installs `orchestrator-routing` as the skill-first routing entrypoint
- adds explicit routing output expectations:
  - `complexity`
  - `role-chain`
  - `next-step`
  - `reason`
- supports custom Codex home targets via `--target`
- supports machine-readable diagnostics with:
  - `codex-agents status --json`
  - `codex-agents doctor --json`
- supports global command linking through:
  - `bin/codex-agents link`
  - `codex-agents unlink`

## Bundled Commands

- `install`
- `update`
- `status`
- `doctor`
- `uninstall`
- `list`
- `version`
- `link`
- `unlink`

## Verification Highlights

This release line has been exercised with repository tests covering:

- install/update managed entrypoint behavior
- orchestrator skill installation
- doctor and status summaries
- target-aware install/status/uninstall roundtrip
- JSON diagnostics and version output
- link/unlink roundtrip, including symlinked `version` execution

## Notes

- After install or update, users should open a fresh Codex conversation so the updated skill list is visible.
- The managed `AGENTS.md` block is the current compatibility layer for making `orchestrator-routing` the default routing preference in Codex.
- `version` currently reports the repo short SHA, which is enough for support and release tracking while the project is still in the `0.x` phase.

## Next Step

The intended next milestone after `v0.1.0` is a Homebrew formula draft using the tagged release tarball and checksum as the packaging source.
